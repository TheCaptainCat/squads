Group a feature's work into a handful of tasks by slice/story, each with subtasks
(`add-subtask`, mapped to a story via `--story USn`) — not one task per finding/piece.

Pierre flagged FEAT-449: I broke it into 9 flat sibling tasks (TASK-450–458); it should
have been ~3–4 grouped tasks (e.g. core / webview / display / meta) with the pieces as
subtasks.

Why: fewer top-level items = a cleaner, more legible board; subtasks carry the granularity
without cluttering the task list. Apply to future breakdowns; don't churn an in-flight one.
