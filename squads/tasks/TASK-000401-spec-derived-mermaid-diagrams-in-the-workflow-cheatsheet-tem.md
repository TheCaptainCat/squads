# Spec-derived Mermaid diagrams in the workflow cheatsheet

Embed auto-generated Mermaid diagrams in the workflow cheatsheet template `src/squads/_rendering/templates/workflow.md.j2` so the cheatsheet shows the shape of the spec instead of only prose describing it. Per ADR-398 the target is this template **only** — not `docs/`, which is static, bundled-vocab, adopter-facing prose and would force hardcoding the built-in types. The cheatsheet is already spec-derived (it loops `spec.items`, `parent_chain`, `linearize_lifecycle`), already regenerated by `sq sync`, embedded in the `squads` skill body, and rendered live by `sq workflow`.

## Scope

Two spec-derived diagrams, both as fenced Mermaid in the template:

1. Item-type **hierarchy flowchart** derived from the parent/child structure already computed in the template via `parent_chain(spec, item_type)` / `spec.items` (the same data behind the existing "Hierarchy:" lines). Emit it as a Mermaid `flowchart` fence.
2. Per-type **lifecycle `stateDiagram-v2`** derived from each type's workflow machine. The template already has `linearize_lifecycle` and `spec.machine_for(type)` available as globals (`src/squads/_rendering/_engine.py`), plus `spec.lifecycles`. A `stateDiagram-v2` wants the actual transition edges, not just a linear ordering — if the transition map is not already reachable from the template globals, expose a small helper in `_engine.py` that yields the machine's transitions (sourced from the status machines in `src/squads/_workflow.py`), rather than hardcoding transitions in the template.

## Constraints (per ADR-398 / EPIC-325 / EPIC-335 / FEAT-334)

- Fully spec-driven: correct for a customized vocab (dropped/renamed/re-prefixed types), never a hardcoded type literal. Same discipline as the existing genericized cheatsheet loops.
- Regenerated by `sq sync` like the rest of the cheatsheet (no separate generation path).
- Degrades to a plain fenced code block in the Rich terminal (`sq workflow`) — the fence should not break terminal rendering; verify what `sq workflow` prints.
- Regenerate `templates_manifest.json` after the template change.

## Tests

Assert the rendered cheatsheet contains the fenced hierarchy flowchart and a per-type lifecycle `stateDiagram-v2`, and that both reflect the active spec (e.g. a renamed/custom type shows up, a bundled literal does not leak). A CLI smoke test that `sq workflow` still renders. Name tests by behavior.
