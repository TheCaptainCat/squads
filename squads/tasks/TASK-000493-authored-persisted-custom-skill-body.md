## Scope

Give a custom (author-defined) skill a real, persistent instructional body, and make the
system/custom distinction visible — implementing ADR-492 Pillars 1 and 2.

### 1. `is_system_skill` predicate (derived, not stored)

Add a pure predicate in `_interactions` (alongside `bundled_skill_slugs`/`custom_skill_slugs`):

```
is_system_skill(slug, spec) := slug in (bundled_skill_slugs() ∪ custom_skill_slugs(spec))
```

It takes the active `WorkflowSpec` so a project that renames/drops a type re-derives cleanly.
Nothing is persisted — this is the single classifier used by both the authoring gate and the
`show` label below.

### 2. Relax the `set_body` meta-guard (`_services/_items.py`)

`set_body` today rejects every meta type except operator ("its body is generated from its
fields"). Relax it to also admit a `skill` item **when that skill is not template-owned** —
i.e. `not is_system_skill(slug, spec)`. A system/bundled skill (`squads`, `greeting`,
`sq-memory`, `sq-<type>`) stays rejected with a clear message (its body is regenerated by sync,
so authoring it would be silently discarded). The slug for the check is the skill's
`extra[slug]` (fallback `item.slug`). Marker-safe edit only (invariant #3), same `:body` region
machinery items already use.

### 3. `sq skill <n> body` verb (`_cli/_skill.py`)

Add a `body` command to the addressed subgroup mirroring the item `body` verb: `-m/--message`
(repeatable), `--file` (`-` = stdin), `--append`. Routes to `svc.set_body(...)`. The
create-time `## Instructions` stub from `agents/skill.md.j2` remains the starting point the
author overwrites.

### 4. `sq skill <n> show` label

Label the skill **system (template-owned)** vs **custom (authored)** in the panel, computed on
the fly via `is_system_skill`. Add the same classification to the `--json` output (e.g. a
`system: true|false` field) for parity.

## Acceptance

- `sq skill body` sets and re-edits a custom skill's multi-paragraph body (message or file);
  the latest edit is what `sq skill show`/`read_body` returns.
- The authored body survives `sq sync`, `sq skill <n> regen`, and `sq repair` unchanged
  (structurally already true — sync/regen never iterate author-defined skills, repair never
  reads bodies; add a test that proves it, do not add a new regeneration path).
- A freshly-added custom skill with no authored body yet still renders a coherent, minimal
  skill file and a valid pointer (no error).
- `sq skill body` on a bundled/system skill is rejected with a clear message.
- `sq skill show` makes system-vs-custom obvious without opening the file.
- Tests: service-level (author/append/persist; reject on system; survives sync+regen+repair)
  and a CLI smoke test (body verb + show label). Name tests by behaviour, not by ticket id.

## Code sites

- `src/squads/_interactions/__init__.py` — `bundled_skill_slugs`, `custom_skill_slugs`, add

- `src/squads/_services/_items.py` — `set_body` meta-guard.
- `src/squads/_cli/_skill.py` — addressed subgroup (add `body`; extend `show`).
