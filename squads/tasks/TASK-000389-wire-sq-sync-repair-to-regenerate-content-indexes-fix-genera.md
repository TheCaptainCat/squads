Carry the amended storage/id decision (per REV-388, option B) into the machinery: keep the committed `.index.jsonl` roll-ups (same posture as `.squads.json` — a rebuildable derived index we commit for convenience) but fix the regeneration wiring and the generated-file stamp so a merge conflict on a committed index has an honest, mechanical, one-command resolution.

## Scope

- **Regenerate every content index on `sq sync`.** Wire the maintenance sync path (`MaintenanceMixin.sync`, `src/squads/_services/_maintenance.py`) to regenerate each content-index `.index.jsonl` — every role's `squads/agents/memory/<role>/` folder and `squads/board/` — whole from that folder's `.md` files, in the same pass that regenerates the other tool-owned managed files. Today the whole-folder regeneration runs only on the add/forget mutation path, so `sq sync` never rebuilds these indexes; sync's "regenerate all tool-owned managed files" contract does not currently hold for them.

- **Correct `GENERATED_STAMP` to match reality.** In the shared generator (`_content_index.py`) set the header `generated` stamp to the exact string the amended decision specifies: regenerated by `sq` on add/forget and `sq sync` — do not hand-edit. Name only the paths that actually regenerate the file — add, forget, and `sq sync` — and drop the phantom "edit" trigger (there is no in-place edit path).

- **Mechanical resolution for a conflicted index.** `sq repair` (or sync) must offer the mechanical fix for a committed index that a merge left conflicted: regenerate it whole from the folder's `.md` files and discard the conflict markers. A conflict on a committed `.index.jsonl` then has a one-command, non-mutating-content resolution (the `.md` source of truth already merged cleanly, so no content is lost — only the derived roll-up is reconciled).

## Notes

- The generator and the committed `.index.jsonl` format are shared by agent memory and the team board, so this fix spans both features: wiring sync/repair to regenerate covers the board's `squads/board/.index.jsonl` as well as each role's memory index. It is authored under the memory feature because the shared generator and store live there.

- This is the resolution path the amended decision documents: distinct adds on two branches leave the `.md` content files merging cleanly (no memory lost), the derived committed index conflicts, and re-running sync/repair regenerates it from the `.md` files to resolve.
